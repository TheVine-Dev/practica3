GIT

  • ¿Qué es git?
  • Git es un sistema de control de versiones distribuido que se utiliza para realizar un seguimiento de los cambios en el código fuente durante el desarrollo de software
  • ¿Para que funciona el comando Git init?
  • El comando git init se utiliza para inicializar un nuevo repositorio de Git en un directorio. Al ejecutar este comando en un directorio que contiene código fuente o archivos que deseas versionar, Git crea un nuevo repositorio local en esa ubicación.
  • ¿Que es una rama?
  • Una rama es una línea de desarrollo independiente que se deriva de la rama principal (generalmente llamada "máster" o "main"). Las ramas permiten a los desarrolladores trabajar en características, correcciones de errores o cualquier modificación del código de manera aislada, sin afectar directamente la rama principal del proyecto.
  • ¿Como saber en que rama estoy?
  • Con el comando “Git Branch” Este comando mostrará una lista de todas las ramas presentes en el repositorio local y resaltará la rama en la que te encuentras actualmente. La rama actual estará precedida por un asterisco (*).
  • ¿Cuales son los comandos más esenciales de Git?

  • ¿Quien creo GIT?
  • Linus Torvalds en 2005, con el propósito de gestionar el código fuente del kernel de Linux y superar las limitaciones de los sistemas de control de versiones existentes en ese momento.
  • ¿Que es Git Flow?
  • Git Flow es un conjunto de reglas y extensiones para Git que se utiliza para facilitar y estandarizar el flujo de trabajo de desarrollo de software.
  • ¿Que es trunk based development?
  • El Trunk Based Development es una metodología de desarrollo de software que se centra en mantener un único tronco principal (también conocido como rama principal, rama "trunk" o "master") como la rama principal del repositorio de código.
